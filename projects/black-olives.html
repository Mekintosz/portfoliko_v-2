Black Olives — Restaurant Page

A responsive, visually rich, static website for a Mediterranean restaurant with table booking UI, newsletter subscription, and smooth, accessible navigation.

HTML5 CSS3 JavaScript Parcel

Project Overview

  • Project name: Black Olives — Restaurant Page
  • Live site: black-olives (GitHub Pages)
  • Repository: github.com/Mekintosz/black-olives
  • One-liner: Responsive static site with booking UI, newsletter, and smooth navigation.
  • Primary users: Prospective diners exploring brand, menu, chef, location, booking.
  • Goals: Communicate brand identity, showcase venue/chef, entry point to booking, capture newsletter signups.

Features

Responsive layout

Mobile-first CSS Grid/Flexbox with media queries; hero image with overlayed navigation; scales up to desktop.

Accessible, smooth navigation

Fixed header with anchor links; mobile menu toggles with aria-expanded/aria-label, closes on Escape; smooth scrolling; body scroll lock when open.

Table booking

Dedicated book-table.html with form validation (future date, closed Mondays, seats range checks).

Newsletter subscription

Inline form for name/email styled consistently with booking; currently posts to #.

Location & contact

OpenStreetMap iframe, prominent contact details, and footer repeats.

Performance & media

WebP images, lazy loading, SVG icons/logos, embedded PDF menu.

Semantic HTML

Landmarks and labeled controls for basic SEO and a11y.

Tech Stack

Frontend

  • JavaScript (ES modules), HTML5, CSS3
  • Module structure: src/index.js imports modules/nav.js and modules/bookingDate.js
  • Styling: single CSS with custom properties and media queries
  • Fonts: Google Fonts (Libre Bodoni, Libre Caslon Text)

Tooling & Build

  • Bundler: Parcel 2 (dev/build)
  • Package manager: npm scripts
  • Deployment: gh-pages package to publish dist

Hosting & Services

  • Static hosting: GitHub Pages
  • Map: OpenStreetMap iframe

Architecture

  • Container: static site served by GitHub Pages
  • Pages: index.html, book-table.html
  • Assets: images (WebP/PNG/SVG), Current-menu.pdf
  • Scripts: index.js initializes modules on DOMContentLoaded
  • Navigation module toggles menu and scroll lock; booking module validates date

Deployment

  • Enable GitHub Pages from gh-pages branch
  • Build with --public-url ./ for relative paths
  • Optional: GitHub Actions to build on main and publish to gh-pages

Development Roadmap

Short-term

  • Smoother mobile nav animation and icon transitions
  • Improve booking layout on small screens
  • Integrate mock/serverless backend for forms
  • Add Playwright smoke tests

Medium-term

  • Backend endpoints with validation and notifications
  • Persist bookings/subscriptions (Supabase/Airtable)
  • Analytics and basic SEO improvements

Non-goals & Future Work

Full backend/CMS/auth are out of scope for now. Future: serverless forms, admin dashboard, CMS, CI, automated image pipeline.
